**Документация по проекту "Книжный сервис"**

**Описание проекта:**

Проект "Книжный сервис" представляет собой веб-приложение, которое позволяет добавлять, удалять и получить список книг. Проект реализует паттерн Repository, чтобы взаимодействовать с базой данных не напрямую, а через репозиторий.

**Модули проекта:**

* `domain`: модуль содержит классы доменной логики, в данном случае - класс книги (`Book`).
* `infra/storage`: модуль содержит реализации хранилищ для книг.
	+ `memory_storage.py`: реализация хранилища в памяти.
	+ `sqlite_storage.py`: реализация хранилища для SQLite.
* `repository`: модуль содержит интерфейс и реализации репозиториев для книг.
	+ `abstract_repository.py`: интерфейс репозитория.
	+ `memory_repository.py` и `sqlite_repository.py`: реализации репозиторий для хранилищ в памяти и SQLite соответственно.
* `service`: модуль содержит сервисы для работы с книгами.
	+ `book_service.py`: сервис для работы с книгами, который использует репозиторий для хранения данных.
* `context`: модуль содержит класс контекста, который используется для получения сервиса и репозитория.

**Классы проекта:**

* `Book`: класс книги, который содержит атрибуты title, author, year и pages_count.
* `MemoryStorage` и `SQLiteStorage`: реализации хранилищ для книг в памяти и SQLite соответственно.
* `AbstractRepository`, `MemoryRepository` и `SQLiteDatabaseRepository`: интерфейс и реализации репозиториев для книг.
* `BookService`: сервис для работы с книгами, который использует репозиторий для хранения данных.
* `Context`: класс контекста, который используется для получения сервиса и репозитория.

**Установка и запуск проекта:**

1. Клонировать репозиторий из Git.
2. Установить зависимости из файла `requirements.txt`.
3. Запустить приложение с помощью команды bookstore.py`.

**API проекта:**

* `/books`: получить список книг.
* `/books`: добавлить новую книгу.
* `/book/<id>`: удалить книгу по id.

**Примечания:**

* Проект реализует паттерн Repository, чтобы взаимодействовать с базой данных не напрямую, а через репозиторий.
* Проект использует Flask для создания веб-приложения.
* Проект использует SQLite для хранения данных.
